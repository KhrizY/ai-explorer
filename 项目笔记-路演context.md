## 一、项目一句话

**「AI 探险号」是一个面向 9-12 岁儿童的公益 AI 科普互动网站**——把晦涩的人工智能原理，做成孩子能"玩着学懂"的探险旅程。

- 形态：零框架的纯前端静态网站（vanilla JS + HTML + CSS），可离线、可预览、易部署；正在扩展为“众包课程创作-上传-归档”系统。
- 当前阶段：**2.0 版本（推倒重来重构）**。1.0 已归档，主线与进阶课程均已接入主阅读器。
- 一句话价值：**让不会数学的孩子，靠直觉和动手玩，真正"看见"AI 是怎么工作的，而不是把 AI 当成魔法。**

---

## 二、为什么做 / 要解决的痛点

1. **AI 已是这代孩子的"空气"**，但市面科普要么太浅（只会喊"AI 很厉害"），要么太深（直接上公式、术语）。
2. 9-12 岁正处于皮亚杰的**具体运算→形式运算过渡期**：能理解抽象，但仍需"看得见摸得着"的锚。市面缺少为这个年龄段量身设计的 AI 课程。
3. 大众对 AI 有两种危险误解：**当成魔法**（盲目崇拜）/ **当成洪水猛兽**（盲目恐惧）。孩子更需要建立**理性、批判、不神化**的第一印象。

**我们的答案**：用游戏化交互 + 教育学锚点，把 AI 从黑箱拆开，一格一格给孩子看清楚。

## 三、核心教育思想（PPT 的"理论支撑"页）

### 1. 课程锚定权威框架：AI4K12「五大理念」
美国 AI4K12 计划提出的 K-12 AI 教育五大支柱，是本项目课程主线的骨架：
1. **感知（Perception）** — AI 如何"看/听"
2. **表示与推理（Representation & Reasoning）**
3. **学习（Learning）** — AI 如何从数据里学
4. **自然交互（Natural Interaction）** — 如何"说人话"
5. **社会影响（Societal Impact）** — AI 的伦理与双刃剑

### 2. 四把「心智桩」（贯穿全站的价值观锚点）
无论学到哪一课，都反复夯实这四个认知，防止把 AI 神化：
- 🧲 **不是魔法**：它只是从大量例子里学规律
- 🍎 **垃圾进垃圾出**：数据差，AI 就差（Garbage in, garbage out）
- 🎭 **它不懂，在猜**：AI 靠模式匹配"猜"，并不真"理解"
- ⚖️ **会犯错，要怀疑**：AI 会出错，要保持批判

进阶架构课再加两个锚：🔧 **它只是在调数字** · 🪜 **一层层往上**。

### 3. 三套教学法融合
- **皮亚杰认知发展论**：面向 9-12 岁，全程**无数学公式**，只取直觉。
- **3Blue1Brown 方法**：先给一个"哇"的动机实例 → 再讲抽象；先建直觉 → 再点原理。
- **多邻国（Duolingo）游戏化**：XP 经验值、连胜、**安全的失败**（答错不用刺眼红色，用暖琥珀鼓励）、吉祥物陪伴、徽章收集。

### 4. 术语"双段制"（由浅入深引导晦涩术语）
划线词点开分两段：`{简：大白话`, `pro：专业一层}`。
- 核心目的：**由浅入深**，不是重复大白话。专业段给"机制/原因"，不是换句话说。
- 中文常用词（如"过拟合"）**不给英文**；英文常用词（**Transformer / CNN / RNN / LLM / Agent**）给**中英对照**释义。
- 没有可补充的就不补充，避免为凑而凑。

## 四、产品结构（PPT 的"产品全景"页）

### 主线：6 章探险（看→学→说→做→信 + 序章）
| 章 | 主题 | 对应 AI4K12 |
|---|---|---|
| 序 | 什么是 AI（破除魔法感） | 总纲 |
| 看 | AI 如何感知（识别图像/声音） | 感知 |
| 学 | AI 如何学习（训练=猜→对答案→调整） | 学习 |
| 说 | AI 如何理解语言 | 自然交互 |
| 做 | Agent 如何自己动手做事 | 交互+推理 |
| 信 | AI 的伦理·双刃剑·如何理性相信 | 社会影响 |

主线共约 35 个知识点，全程直觉化、糖果化，阿波（吉祥物）带路。

### 进阶：14 门独立专题课（"再深一层"，初中+/跨龄）
平铺在主页 6 课下方，相互独立。三类：
- **架构探秘**：神经网络在算什么 / RNN 与记忆 / 注意力的数学 / Transformer 全貌 / 扩散模型画画 / 大模型训练三部曲 / 强化学习 / 经典机器学习…
- **AI 大观**（思辨类）：苦涩的教训（Bitter Lesson）/ AI 简史 / AI 与社会 / AGI 与对齐…
- **动手实践**：2 门。

进阶课 vs 主线差别：受众更大、可到"机制/数学的直觉版"、3B1B 风格、术语更多、复用可视化 demo。

> 2026-07 决策：CNN 进阶课已移除，因为主线「看」章已经覆盖卷积/滑窗/特征直觉；若未来恢复，必须讲 padding/stride、感受野、多特征图等真正新增内容。
> 当前 14 门里保留「经典机器学习：KNN/聚类/阈值」，它服务的是非神经网络方法与可解释直觉，不与主线「看」重复。

### 众包课程系统（新增方向）
项目正在从“单一课程网站”扩展为“可众包的互动科普生产系统”：
- 贡献者下载/复制创作 skill，按固定 JSON 格式生产课程。
- `upload.html` 提供上传页：导入 JSON、校验、保存到本地实验区、发布到服务器生成 4 位课程码。
- 主站首页有“实验课程”隐藏入口：输入课程码后动态接入现有阅读器。
- 课程包只允许 JSON，不允许上传 JS；互动通过白名单组件 `ux.component + props` 渲染。
- 众包课程长度契约已调整为：每个 C 章节 **4-8 个 S 小节**，推荐 5-6 节；上传页、主站读取和发布 API 都会校验，避免生成“短小概念卡”而不是完整课程。
- 当前动态 MVP：保存只进入同源浏览器 localStorage；点击发布后由 Node API 写入服务器临时存储并生成不重复 4 位课程码，其他设备在有效期内可访问。
- 下一步生产版：把当前 `.data` 文件存储替换为 KV/Redis/Postgres TTL；过期服务器清除，本地草稿仍保留，可再次发布。

### 吉祥物「阿波（Apo）」
一个好奇、爱鼓励、会自嘲的 AI 讲解机器人，全程带路。2026-07-15 已从"文本口吻"升级为**每个 S 小节开头的形象化讲解员**：机器人弹出语句气泡，文字快速逐字出现，带漂浮、眨眼、天线、挥手、嘴巴动效；复习/减少动效模式直接显示全文。

## 五、交互与 UI 系统（PPT 的"体验设计"页）

### UI 系统「星空实验室 Cosmos-Lab」——双表面模型
融合两种美学：**多邻国的活力** × **深空探险的沉浸**。
- 🌌 **暗色星空面（.cosmos）**：用于地图、旅行框架、通关庆祝——营造探险感。
- 📄 **浅色工作台面（.workbench）**：用于阅读、交互、答题——保护视力、专注学习。
- 6 站主题色（序/看/学/说/做/信各一色 `--st0..--st5`）。
- 反馈色遵循儿童安全：对=绿、错=暖琥珀（**绝不用刺眼红**）、奖励=金。

### 游戏化玩法调研（用户点名要参考的基础玩法）
主线/通关挑战复用了这些经典游戏机制：
- **QTE 动作判定**（动作游戏）——加明显的成功/失败反馈
- **长按滑动条控制**（星露谷钓鱼）
- **方向判定点击 / 横移+向下抓取**（黄金矿工→改成抓娃娃机模式）
- 选项卡拖拽、记忆翻牌、节奏点击、定时点击等
- 一个可复用游戏引擎 `buildArcadeGame(cfg,onWin,onReset)`（浅色舞台、落物接取）
- **UX Demo 库**：28 个可玩演示（神经元点火、前向传播、卷积、池化、注意力连线、扩散去噪、梯度下山、损失曲线、聚类、KNN、分词、RLHF…），既是教具也是游戏素材。

### 奖励系统
XP 经验值 + 金币（获得时有"蹦出→飞向金币卡"动画）+ 徽章收集册。金币收集做成**一次性**（重玩不重复领），通关 +50 只在首次。

## 六、用户（你）提出的关键要求 / 决策（PPT 幕后·迭代逻辑）

> 这一节保留"你说过什么"，方便路演讲"我们是如何被真实需求驱动迭代的"。

**方向决策**
- 目标受众定为 **9-12 岁**；2.0 **推倒重来**；允许**联网调研**做参考；**先预览**再落地。
- 认可四步主线（后扩为 6 章，增加"做/Agent"章）；设计成就/徽章系统。
- 吉祥物想法已进入 2.0：阿波以机器人 + 语音气泡形态出现在每小节开头；后续继续扩展答对/答错/通关状态。

**内容质量要求**
- 术语双段制的多次打磨：专业段不要和大白话重复、不要太口语、30-50 字、去掉"进阶"提示符前缀、中文常用词不加英文括号。
- 每小节可适当出现一两个专业名词，把较专业的解释藏在划线词展开条目里。
- 想深入讲的高阶主题：RNN、自注意力/多头注意力/缩放点积注意力、位置编码、编码器-解码器堆叠、**Bitter Lesson**、Agent 的 CLI 运行原理等——先融进现有体系，再开专题课。要求给出**参考佐证**（如 AI4K12）。

**交互质量要求**
- 每个交互都要有**重置**按钮、要能**取消**；有判断的交互要有明显的**成功/失败反馈**；答题类交互满足要求给明确"完成"提示。
- **"完成"判定不要太快**——应尽量体验过一部分互动才算完成再给奖励（最新一条 git commit 的要求）。
- 避免交互时产生**文字框选**干扰（`.noselect`）。
- 训练机演示要能打到 **100%**，增量做成**线性递减**。
- 拟合图像的散点不要**定死 3 个点**（扩到 11 点）。
- **游戏性仍偏弱**，是后续重点——"发挥创意、提高兴奋感"，答题类互动别太多。

**工程/协作**
- git 环境受限：无 SSH、GitHub 间歇被墙，用 token 走 HTTPS push。用户允许当前 token 暂时保留，后续继续用于推送。
- **当前工程状态**：`index.html` 是主应用入口，主线 6 章 + 14 门进阶课可点击进入、逐段学习、通关收徽章；`upload.html` 是众包课程上传入口。后续可直接迭代主阅读器、章节数据和课程包系统。
- **众包课程决策**：课程包只带 JSON，不带 JS；互动函数由主站白名单组件隔离渲染。当前已支持 `labSlider`、`labChoice`、`labSort`、`labSteps/uxTimeline`、`labMatch`、`labReveal`、`labFlow`。
- **众包课程长度规范**：创作 skill、上传页校验、主站读取校验和发布 API 已统一要求每个 chapter 有 4-8 个 S 小节，解决自动生成课程偏短偏少的问题；上传模板已扩成 4 节示范课。
- **上传页 UX 决策**：流程按“下载创作 Skill → 导入/编辑 JSON → 校验/保存 → 实验区发布/预览”拆成独立面板；工程方案用轻量弹窗解释，不跳长文档。
- **动态发布路线**：localStorage 只负责作者本地草稿；跨设备预览走 API 发布。发布动作生成 4 位课程码和有效期，到期服务器自动清除；作者本地仍保留草稿，可再次发布。实验区支持删除：草稿删除本地记录，已发布课程同时请求服务器清除课程码。
- **等级/勋章系统**：左上角火箭区成为“等级与进度”入口。等级由金币驱动，顶部进度条显示距离下一等级的进度；勋章墙从只统计主线升级为统计主线 + 14 门进阶课 + 已加载实验课程。
- **调试后台**：连续点击金币组件 5 次可进入全解锁调试模式，所有章节/S 卡片/beat 默认展开，方便快速检查页面和交互。
- **进阶 UX 重做状态**：已把待重做的 18 个正式课交互临时下线为“互动可视化施工中”占位，保留正文和术语；新方案继续在 `demos/advanced-ux-compare.html` 测试。

## 九、可直接用作 PPT 大纲的一条线（建议）

1. **痛点**：AI 是这代孩子的空气，但科普非浅即深，且易神化/妖魔化。
2. **定位**：9-12 岁公益 AI 科普互动网站，玩着学懂。
3. **理论底座**：AI4K12 五大理念 + 四把心智桩 + 皮亚杰/3B1B/多邻国三教学法。
4. **产品全景**：6 章主线 + 14 门进阶课 + 吉祥物阿波 + 28 个交互 demo + 众包课程实验区。
5. **体验设计**：星空实验室双表面 UI + 经典游戏玩法 + 安全失败的奖励系统。
6. **差异化**：不神化 AI（心智桩）、无数学直觉化、术语由浅入深双段制。
7. **现状与规划**：2.0 主线、进阶课与众包课程 MVP 已可玩；下一步加厚进阶课、做动态发布 API、继续增强吉祥物和游戏性。
8. **愿景**：让每个孩子都对 AI 有理性、好奇、不盲从的第一印象。
