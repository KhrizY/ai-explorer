# 🔗 注意力的数学 · 内容设计

> 进阶课（架构探秘，初中+）· 8 知识点 · 复用 `attention` demo，是 Transformer 的地基。
> 目标：讲清那句著名的 "Attention is all you need"——**AI 读句子时，是怎么决定"该重点看哪几个词"的**。
> 心智锚点：🔦 它在分配注意力 · 🎭 它不懂在猜（靠相关度打分，不是真理解） · 🪜 一层层往上。
> 前置：[[神经网络在算什么]]（懂"打分/加权和"就够）。

**一句话钩子**："小明把苹果给了小红，因为**她**饿了。"——这个"她"指谁？你一秒就知道是小红。AI 读到"她"时，也要在一整句话里**回头找**它到底指谁。它靠的不是语法书，而是一套会"打分"的机制：**注意力（Attention）**。这门课我们把这套打分拆开看。

---

## A1 · 难题：词的意思要看上下文
- **主要矛盾**：同一个词，在不同句子里意思天差地别，AI 怎么办？
- **直觉/类比**："苹果"在"我吃苹果"里是水果，在"苹果发布会"里是公司。词本身不够，得**看它周围有哪些词**。
- **可视化 + UX 互动方案**：
  - 复用/新做小面板 `wordctx`：一个高亮词"苹果"，下面两个句子卡片可点击切换（🍎水果／📱公司）；切换时，句中和"苹果"相关的词自动连出高亮线，右侧插画随之变。
  - **玩法**：点句子 → 看"苹果"被哪些邻词"拉"向不同含义。
  - **成功反馈**：两句都点过 → "✅ 你发现了：同一个词，看邻居定意思！"
- **讲解文本**：AI 一开始给每个词一个"通用意思"，但真正读句子时，得让每个词**看看周围的词**，再更新自己的含义。这个"互相看一看、彼此影响"的动作，就是注意力要干的事。
- **术语**：[[上下文]]｜简：一个词周围的其它词。｜pro：context，同一个词在不同上下文里向量表示会不同，这是注意力存在的理由。

## A2 · 核心动作：谁该看谁（相关度打分）
- **主要矛盾**：句子里词那么多，每个词该"重点看"哪几个？
- **直觉/类比**：读到"她"，你会给句中每个词打个"相关分"：小红→高分、苹果→低分、给→低分。谁分高，就多看谁一眼。
- **可视化 + UX 互动方案**：
  - 复用 `attention` demo（词与词之间的注意力连线，线越粗=相关度越高）。
  - **玩法**：点选句中一个词（如"她"），其它词射出**粗细不同的连线**指向它，粗线=高相关。可拖动一个"聚焦"词依次看每个词的注意力分布。
  - **成功反馈**：把"她"连到正确的"小红"（最粗线指对）→ 撒花 + "🔦 注意力对准了！"
- **讲解文本**：注意力的核心就一句话：**每个词，对句子里所有词打一个"我该多关注你几分"的分数**。分高的多看，分低的少看。粗线连着的，就是它认为最相关的词。
- **术语**：[[注意力]]｜简：让每个词"重点关注"句中最相关词的机制。｜pro：attention，为每对词算相关度权重，再据此加权融合信息。

## A3 · Q / K / V：三副面孔
- **主要矛盾**：这个"相关分"到底怎么算出来的？
- **直觉/类比**：给每个词发三张卡：
  - **Query（问）**="我在找什么"（"她"在找它指代的人）
  - **Key（答）**="我能提供什么"（"小红"标明"我是个女生名字"）
  - **Value（货）**="我真正携带的信息"。
  用"她"的**问卡**去比对每个词的**答卡**，越匹配分越高；再按分数把大家的**货卡**加权收集起来。
- **可视化 + UX 互动方案**：
  - 新做 `qkv` 小演示：每个词显示三张小卡 Q/K/V。拖某个词的 **Q 卡**去碰其它词的 **K 卡**，匹配度以发光强弱显示；匹配后对方 **V 卡**按亮度"流"进来汇合。
  - **玩法**：给"她"的 Q 找最亮的 K。
  - **成功反馈**：Q-K 配对正确 → "🗝️ 钥匙对上锁了！"
- **讲解文本**：每个词其实同时扮演三个角色：**问（Query）**、**答（Key）**、**货（Value）**。拿一个词的"问"去对所有词的"答"，算出相关分；再按分数，把大家的"货"打包带走。这就是注意力的全部机关。
- **术语**：[[Query／Key／Value]]｜简：词的三种角色——找什么、能答什么、带什么信息。｜pro：Q/K/V 都是词向量乘不同权重矩阵得到的三个向量。

## A4 · 缩放点积：打分的公式（直觉版）
- **主要矛盾**：Q 和 K 怎么变成一个"分数"？
- **直觉/类比**：把 Q 和 K 两个向量做**点积**——方向越一致、分越高（像两个箭头越同向，投影越长）。词多时分数会变很大，所以再**除以一个数"缩放"**一下，别让它爆表。
- **可视化 + UX 互动方案**：
  - 新做 `dotprod` 小演示：两个可旋转的箭头 Q、K，实时显示点积值（同向→大、垂直→0、反向→负）；旁边一个"缩放"滑块，拖动看分数被压回合理范围。
  - **玩法**：转箭头让分数最大 / 拖缩放让分数不爆。
  - **成功反馈**：箭头调到同向 → "📈 方向越一致，越相关！"
- **讲解文本**：算相关分的办法叫**点积**——两个词的向量方向越一致，分数越高。词一多，这些分容易变得特别大，所以要**除以一个数缩放**，让它们保持温和。这就是论文里 "缩放点积注意力" 的名字由来。
- **术语**：[[缩放点积]]｜简：用点积算相关度、再除一个数防止过大。｜pro：scaled dot-product attention，Q·Kᵀ 后除以 √d_k，是 Transformer 的注意力公式。

## A5 · Softmax：把分数变成"注意力比例"
- **主要矛盾**：一堆大小不一的分，怎么变成"该分多少注意力"？
- **直觉/类比**：把所有相关分丢进一个"归一化机器"（Softmax），出来变成加起来=100% 的**比例**：小红 70%、苹果 5%、给 3%……
- **可视化 + UX 互动方案**：
  - 复用 `attention` demo 或加一个**注意力条形图**：拖高某个词的原始分，看下方比例饼/条实时重新分配（此消彼长，总和恒 100%）。
  - **玩法**：把注意力"喂"给正确的词，让它占比最高。
  - **成功反馈**：正确词占比 >60% → "🎯 注意力集中！"
- **讲解文本**：原始分数有高有低，还有负的，不好直接用。Softmax 把它们变成**加起来正好 100% 的比例**——高分的拿大头，低分的拿零头。这就是"注意力"真正分配出去的样子。
- **术语**：[[Softmax]]｜简：把一组分数变成加起来=100% 的比例。｜pro：softmax，注意力权重归一化步骤，指数放大差距、突出最相关项。

## A6 · 多头注意力：同时看好几种关系
- **主要矛盾**：一句话里，词之间不止一种关系（谁指谁、谁修饰谁、谁是动作对象……），一套注意力看得过来吗？
- **直觉/类比**：派**好几组注意力**同时读同一句话，各盯一种关系：一组管"代词指谁"，一组管"形容词修饰谁"，一组管"主谓宾"。最后把各组结论汇总。像一群侦探各查一条线索，再碰头。
- **可视化 + UX 互动方案**：
  - 新做 `multihead` 小演示：同一句话叠 2-4 层不同颜色的注意力连线（每个"头"一种颜色），可用开关逐个点亮，看每个头关注的关系不同。
  - **玩法**：猜"这个头在管什么关系"，点开验证。
  - **成功反馈**：认出某个头的分工 → "🕵️ 这个头专管代词！"
- **讲解文本**：一套注意力只能盯一种关系太浪费。于是派**好几组（好几个"头"）**同时读句子，各看各的关系，最后合并。这就是**多头注意力**——多几双眼睛，理解更全面。
- **术语**：[[多头注意力]]｜简：多组注意力并行、各看一种词间关系，再合并。｜pro：multi-head attention，把 Q/K/V 拆成多个子空间并行做注意力，增强表达力。

## A7 · 位置编码：告诉 AI 词的先后
- **主要矛盾**：注意力同时看所有词，它其实**不知道词的顺序**！可"猫追狗"和"狗追猫"意思完全不同。
- **直觉/类比**：给每个词额外别上一个"**座位号**"，AI 就知道谁在前谁在后。座位号用一种波浪式的编码，让"第3个"和"第4个"既不同又有规律。
- **可视化 + UX 互动方案**：
  - 新做 `posenc` 小演示：一排词，每个带一个彩色"位置波纹"标签；拖动打乱词序，右侧句意插画随之改变（猫追狗↔狗追猫），凸显顺序的重要。
  - **玩法**：交换两个词的位置，看意思翻转。
  - **成功反馈**：体验到"换位=换意" → "🔢 顺序也是信息！"
- **讲解文本**：注意力有个大 bug：它一次看全部词，**天生不分先后**。但"猫追狗"和"狗追猫"完全两码事。解决办法是给每个词加一个**位置编码**——相当于别上座位号，让 AI 知道词的顺序。
- **术语**：[[位置编码]]｜简：给每个词加上"它排第几"的信息。｜pro：positional encoding，用正弦波或可学习向量注入顺序信息，弥补注意力的位置无关性。

## A8 · 为什么它这么强：并行 + 看全局
- **主要矛盾**：注意力凭什么打败了以前的老办法（RNN 逐词读）？
- **直觉/类比**：老办法像**排队传话**，一个词一个词地读，长句子前面容易忘；注意力像**全班同时开会**，任何两个词都能直接对话，再远也能一眼看到。而且大家同时算，**又快又不健忘**。
- **可视化 + UX 互动方案**：
  - 新做对比 `rnn-vs-attn`：左"排队传话"（信息逐格传，末端变淡=遗忘）／右"全员连线"（首尾直接连线，无衰减）。同放一遍，直观看差别。
  - **玩法**：拉长句子，看左边末端"忘光"、右边依然清晰。
  - **成功反馈**：看懂差别 → "⚡ 又快又记得住！"
- **讲解文本**：以前的模型逐词读，句子一长，前面就忘了（[[RNN 与记忆]] 里细讲）。注意力让**任意两个词直接连线**，再远也能互相看到，而且所有词**同时**算——又快又不健忘。这就是它一举取代旧方法的原因。
- **术语**：[[并行]]｜简：所有词同时计算，不用排队。｜pro：parallelism，Transformer 可并行处理全序列，是它能训到超大规模的关键工程优势。

## 收束
注意力的秘密：**每个词用"问卡"去对所有词的"答卡"打相关分（缩放点积）→ Softmax 变成注意力比例 → 按比例收集大家的"货"**。多派几组（多头）看不同关系，再别上座位号（位置编码）补上顺序。它让任意两词直接对话、还能并行——于是有了后面的 Transformer。

- 往下接：这套注意力，堆起来就是 [[Transformer 全貌]]。
- 和主线呼应：主线「说」章"AI 读一句话会重点看某些词"，这门课把"重点看"拆成了可算的分数。
