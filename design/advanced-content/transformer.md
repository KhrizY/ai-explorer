# 🤖 Transformer 全貌 · 内容设计

> 进阶课（架构探秘，初中+）· 8 知识点 · 复用 `attention`/`forward`/`token` demo。
> 目标：把驱动 ChatGPT 的那个架构——**Transformer**——从"神秘缩写"变成一条看得懂的流水线。
> 心智锚点：🪜 一层层往上 · 🎭 它在猜下一个词 · 🔧 只是在调数字。
> 前置：[[注意力的数学]]（必须先懂 Q/K/V 打分）、[[神经网络在算什么]]。

**一句话钩子**：ChatGPT 里的 "T" 就是 **Transformer**。它不是一个魔法大脑，而是一条**流水线**：把你的话切成词 → 变成数字 → 让词互相"开会"（注意力）→ 叠很多层 → 最后**一个词一个词地猜**接下来该说什么。这门课我们从头到尾走一遍这条流水线。

---

## T1 · 分词：先把话切成"积木"
- **主要矛盾**：AI 不认字，怎么读你的句子？
- **直觉/类比**：先把句子切成一小块一小块的"**词元（token）**"——有时是一个词，有时是半个词。像把一句话拆成乐高积木。
- **可视化 + UX 互动方案**：
  - 复用 `token` demo（输入一句话，实时切成彩色小块，显示块数）。
  - **玩法**：输入自己的名字/一句话，看它被切成几块；试试长英文单词被切成半块。
  - **成功反馈**：切出 token → "🧱 你的话变成了 N 块积木！"
- **讲解文本**：AI 读句子的第一步是**分词**：把话切成一个个小块，叫"token"。有的 token 是完整词，有的是词的一部分。之后每一步，AI 都在跟这些积木打交道。
- **术语**：[[词元]]｜简：句子被切成的最小处理单位。｜pro：token，可能是词、子词或字符，模型的输入/输出都以 token 计。

## T2 · 词嵌入：把积木变成数字坐标
- **主要矛盾**：token 还是"字"，神经网络只吃数字，怎么转？
- **直觉/类比**：给每个 token 一串数字当"**坐标**"，把它放进一个"意思地图"里——意思相近的词（猫/狗）坐得近，无关的词（猫/汽车）离得远。
- **可视化 + UX 互动方案**：
  - 复用/新做 `embed` 小演示：一张 2D "意思地图"，散布词点；点一个词高亮它的最近邻（猫→狗/兔近，→汽车远）。
  - **玩法**：拖一个新词进地图，猜它该落在哪一片。
  - **成功反馈**：放对区域 → "🗺️ 意思相近，坐得就近！"
- **讲解文本**：每个 token 被变成一串数字，叫**词嵌入**，相当于它在"意思地图"上的坐标。意思像的词坐得近。有了坐标，神经网络就能计算它们了。
- **术语**：[[词嵌入]]｜简：把词变成一串代表其含义的数字。｜pro：embedding，把 token 映射到高维向量空间，语义相近则向量相近。

## T3 · 加座位号：位置编码
- **主要矛盾**：注意力不分先后，得补上顺序（承接注意力课）。
- **直觉/类比**：给每个词的坐标再**加上"座位号"**，AI 才知道谁先谁后。
- **可视化 + UX 互动方案**：
  - 复用 [[注意力的数学]] 的 `posenc` 演示（词序打乱→句意翻转）。
  - **玩法**：交换语序看意思变。
  - **成功反馈**：见证"换序换意" → "🔢 顺序也进坐标了！"
- **讲解文本**：接着给每个词的坐标别上**位置编码**（座位号），补上"谁在前谁在后"。现在每个词都带着"我是谁 + 我排第几"进入下一步。
- **术语**：[[位置编码]]｜简：给词加上排序信息。｜pro：positional encoding，与词嵌入相加后送入注意力层。（详见 [[注意力的数学]]）

## T4 · 一个 Transformer 块：注意力 + 前馈
- **主要矛盾**：核心的"一层"到底在干什么？
- **直觉/类比**：每一层做两件事：① **注意力**——让词互相开会、更新理解；② **前馈网络**——每个词各自再"想一想"、加工一下。开会 + 独立思考，交替进行。
- **可视化 + UX 互动方案**：
  - 新做 `block` 结构演示：一个块里两格——先"注意力"（词间连线亮起）→再"前馈"（每个词单独过一个小神经元，复用 `forward` 脉冲）。点"运行这一层"看数据流过两格。
  - **玩法**：单步走完一层的两个阶段。
  - **成功反馈**：走完一层 → "🔄 开会 + 思考，完成一层！"
- **讲解文本**：Transformer 的一层（一个"块"）就干两件事：先**注意力**让所有词互相看、更新含义；再**前馈网络**让每个词单独深加工一下。这两步搭配，就是最小的处理单元。
- **术语**：[[前馈网络]]｜简：每个词各自再过一遍的小神经网络。｜pro：feed-forward network，逐位置独立的两层全连接，负责非线性变换。

## T5 · 叠很多层：越叠越懂
- **主要矛盾**：一层不够聪明，怎么办？——像神经网络一样，**叠**。
- **直觉/类比**：把这样的块**叠几十上百层**：底层看词与词的近关系，越往上看的关系越抽象（从"这词修饰谁"到"整段在讲什么"）。层层加工，理解越来越深。
- **可视化 + UX 互动方案**：
  - 新做 `stack` 演示：竖着叠 N 个块，点"往上一层"看句子表示逐层"变浓"；侧边标注每层大致在抓什么（词法→句法→语义）。
  - **玩法**：滑动层数，观察理解深度变化。
  - **成功反馈**：叠到顶层 → "🪜 层层堆叠，理解成型！"
- **讲解文本**：一个块不够，就把它**叠很多层**。底层理解词与词的小关系，高层理解整句甚至整段的大意思。"深度学习"的深，在这里就是**层数**。大模型动辄几十上百层。
- **术语**：[[堆叠]]｜简：把很多相同结构的层摞起来。｜pro：stacking，深层 Transformer 靠残差连接与归一化才能稳定训练。

## T6 · 编码器 vs 解码器：读懂 vs 写出
- **主要矛盾**：Transformer 有两种"半边"，分工不同。
- **直觉/类比**：**编码器**擅长"读懂"整段（像做阅读理解），**解码器**擅长"往下写"（像续写作文）。翻译模型两个都用；ChatGPT 这类聊天模型主要用**解码器**——因为它的活是"接着往下说"。
- **可视化 + UX 互动方案**：
  - 新做 `encdec` 对比图：左编码器（双向看全句，箭头互指）／右解码器（只能看左边已生成的词，箭头单向）。点两种模式看信息流方向差别。
  - **玩法**：切"读懂/续写"两种模式，看谁能看到哪些词。
  - **成功反馈**：理解分工 → "📖 一个读、一个写！"
- **讲解文本**：Transformer 有两种搭法：**编码器**双向读、擅长理解整段；**解码器**只能看已经写出的部分、擅长一个词一个词往下写。翻译两个都要；聊天机器人主要靠解码器"续写"。
- **术语**：[[编码器／解码器]]｜简：一个负责读懂、一个负责往下写。｜pro：encoder/decoder，解码器用带掩码的注意力保证"只看过去、不偷看未来"。

## T7 · 它怎么"说话"：一次猜一个词
- **主要矛盾**：AI 到底怎么把回答"写"出来的？
- **直觉/类比**：它其实只会一件事：**猜下一个词**。写完一个词，就把它接到句尾，再猜下一个，如此接龙，直到写完。像玩"词语接龙"停不下来。
- **可视化 + UX 互动方案**：
  - 新做 `nextword` 演示：给半句话，AI 列出候选下一个词 + 各自概率条；点"选中"把词接上去，循环生成，直观看到"接龙"过程。
  - **玩法**：手动帮它选下一个词，看句子长出来；对比高概率/低概率选择带来的不同句子。
  - **成功反馈**：接出一句完整话 → "🔮 一个接一个，猜出整句！"
- **讲解文本**：这么复杂的架构，最后只干一件事：**猜下一个词**。它给每个可能的词打个概率，挑一个接上，再猜下一个……一路接龙写出整段话。所以本质上，它是个超级厉害的"文字接龙机"——**它在猜，不是真懂**🎭。
- **术语**：[[自回归生成]]｜简：一次生成一个词、再拿它继续生成下一个。｜pro：autoregressive generation，每步基于已生成序列预测下一 token 的概率分布。

## T8 · 为什么它改变了一切：规模
- **主要矛盾**：这架构好在哪，值得全世界都用它？
- **直觉/类比**：因为它**能并行、可以无限叠大**（回顾注意力课的"全员开会"）。喂给它越多数据、堆越多层，它就越聪明——而且几乎没撞到天花板。这正应了那句"**苦涩的教训**"：与其手工设计聪明规则，不如把简单结构做大、喂海量数据。
- **可视化 + UX 互动方案**：
  - 新做 `scaling` 演示：一根滑块"模型规模"，拖大 → 能力条随之上涨、涌现出新技能图标（翻译/写代码/推理）。凸显"越大越强"。
  - **玩法**：把规模拖到不同档位，看解锁的能力。
  - **成功反馈**：拖到大规模 → "📈 越大越强——这就是大模型！"
- **讲解文本**：Transformer 最厉害的一点是**能做大**：可以并行计算、可以无限叠层、吃得下海量数据。于是人们发现——把它做得越大、喂得越多，它就越强，还会"涌现"出没教过的新本事。这就是今天所有大模型的共同秘密。
- **术语**：[[规模化]]｜简：把模型和数据做大，能力随之变强。｜pro：scaling，参数量/数据量/算力增大带来性能提升与能力涌现，呼应 [[苦涩的教训]]。

## 收束
Transformer 是一条流水线：**分词 → 词嵌入 + 位置编码 → 叠很多个"注意力 + 前馈"块 → 一次猜一个词往下写**。它能并行、能叠大，于是催生了今天的大模型。记住：再强，它也只是在**猜下一个词**🎭。
