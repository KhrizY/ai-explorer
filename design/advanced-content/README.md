# 进阶课程 · 内容设计规范

> 主线 6 章讲"直觉入门"（9-12 岁）；进阶 14 门讲"再深一层"（初中+/跨龄）。
> 本目录放每门进阶课的**内容设计**（知识点→直觉→交互→讲解文本→术语），供 app 侧按需做成可玩章节。
> **分工**：内容设计在 `design/advanced-content/`；app 游戏/UI 实现在 `index.html`（另有 agent 负责），两边解耦。

## 一、进阶课 vs 主线的差别
| 维度 | 主线（看/学/说/做/信） | 进阶（14 门） |
|---|---|---|
| 受众 | 9-12 岁 | 初中+ / 跨龄（AI大观可低龄） |
| 深度 | 只取无数学直觉 | 可到"机制/数学的直觉版"，仍不硬推公式 |
| 风格 | 阿波带路、糖果化 | 3Blue1Brown 式：先视觉直觉→再点一层原理 |
| 术语 | 少而精、双段 | 多一些真术语，**英文常用词给中英对照** |
| 交互 | 简单可玩 | 复用可视化 demo（neuron/forward/grad/loss/diffusion…）+ 新玩法 |

## 二、每门课的内容结构（模板）
```
课（8-9 知识点）
├─ 一句话钩子（先抛"咦?"）
├─ 逐知识点：主要矛盾(难点) → 直觉/类比 → 交互(复用哪个demo) → 讲解文本(2-4句) → 术语(双段·中英)
├─ 心智锚点沿用 🧲🍎🎭；架构类可加新锚：🔧"它在调数字"、🪜"一层层"
└─ 收束：把这门课和主线/其它进阶课挂钩
```

## 三、写作沿用的两把 skill
- `explain-skill`：先现象后原理、术语双段、诚实边界、UX 先行。
- `viz-ux-skill`：交互"到位"五特征、复用 demo、浅色舞台。

> 已复用现成 demo 之外，注意力+Transformer 两课提出以下新小演示（都遵循浅色舞台 + 重置 + 明显成功反馈）：
