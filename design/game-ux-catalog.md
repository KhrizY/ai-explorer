# 🎮 Game-UX 调研映射库 · 把游戏机制变成教学交互

> 目的：把全网成熟的**基础游戏机制**抽象成可复用的「game-ux」——既能当**讲解交互**，也能当**节末检测游戏**。加新内容时从这里挑一个套用即可。

---

## 一、基础游戏机制总表（全网调研）

来自休闲/超休闲游戏的核心"动词"（玩家已会、零学习成本）+ 你点名的三个经典：

| 机制 | 输入动词 | 代表游戏 | 手感特点 |
|---|---|---|---|
| 计时点击 QTE | 在正确时机点 | 动作游戏判定、节奏大师 | 反射、瞬间、强判定 |
| 长按滑条控制 | 长按/松开，保持在移动区间 | **星露谷钓鱼** | 张力、平衡、持续微调 |
| 方向蓄力抓取 | 看准摆动时机点击 | **黄金矿工** | 瞄准、时机、赌一把 |
| 拖拽分拣 | 拖放归类 | 三消/分类整理 | 满足、有序、可试错 |
| 滑块探索 | 拖动看实时变化 | 各类模拟器 | 因果直观、探索 |
| 连点蓄力 | 快速重复点击 | 敲击类 | 发泄、进度感 |
| 扫描/瞄准移动 | 拖动准星扫过 | 射击/找物 | 探索、发现 |
| 滑动方向 | 上下左右划 | Temple Run | 连续、紧张 |
| 二选一快判 | 极速二选一 | **WarioWare** | 反射、心流 |
| 匹配/合成 | 找相同配对 | 三消/合成 | 模式识别、爽 |

**WarioWare 微游戏心法（用于所有检测游戏）**：① 一局 **4 秒**；② **一句话指令**（"抓！""躲！""分！"）；③ 制造**心流**（难度略高于无聊、略低于挫败）；④ **伪装重复**（同一机制换皮，练很多遍不觉腻）；⑤ **递增难度 + 随机**（越来越快、目标越来越小）。

## 二、Game-UX 映射表（机制 → AI 教学用途）★核心

> 关键洞察：**分类、找规律、时机、平衡** 本就是 AI 的核心动作——游戏机制天然能演示它们。

| Game-UX | 教学用途（演示什么 AI 概念） | 用在哪一章节 | 反馈基调 |
|---|---|---|---|
| **拖拽分拣** | **分类**是 ML 的灵魂：把数据拖进正确类别；把信息拖进"能说/不能说" | 学 S1 认水果 · 信章 隐私分拣 | 对=撒花，错=暖色鼓励 |
| **手摇连点/滑块** | **训练循环**：连点推进"喂→猜→调整"，看准确率爬升 | 学 S2 训练循环 ⭐ | 准确率爬升即正反馈 |
| **滑块探索** | **因果直观**：拖训练程度/数据量，实时看结果变化（表情😵→😻） | 学 S2/S3 | 即时联动 |
| **长按滑条控制（钓鱼式）** | **平衡/调参**：把"准确率"稳定控制在高位区间，手一松就掉——体会调参之难 | 学 S2 进阶 · 数据平衡 | 进度条填满=通关 |
| **方向蓄力抓取（黄金矿工式）** | **抓取正确样本**：摆动的抓钩瞄准，只抓"干净数据"，抓到脏数据会扣分 | 学 S3 数据质量检测 | 抓对+分，抓错整蛊 |
| **计时点击 QTE** | **抓住时机**：在"特征闪现/猜对的瞬间"点击命中 | 通用检测游戏 | 命中撒花 |
| **二选一快判（WarioWare式）** | **快速判断**："魔法还是学习""真话还是编的"极速判定 | 序章 · 说章 幻觉 | 连对加速+连击 |
| **接龙点选** | **猜下一个词**：点候选词把句子接下去 | 说 S1 接龙预言家 | 接顺了给爽感 |
| **是非树点击** | **推理/决策**：逐层点是非题，AI 反猜你想的东西 | 学 S5 决策树 | 猜中揭晓 |
| **扫描滑窗** | **卷积找特征**：拖小窗口扫过图，扫到边缘就亮 | 看章 CNN 直觉 | 叮地亮起 |
| **匹配配对** | **模式识别**：把相似特征配成对 | 看章 · 通用 | 配对消除 |

## 三、复用规范（调用契约）

每个 game-ux 做成**预制组件**，统一：
1. **外壳**：讲解用套 `.ixbox`，检测游戏用套 `.gameframe`。
2. **一句话指令**：开局顶部一句（WarioWare 式），如"👆 把苹果拖进左边筐！"。
3. **参数化**：题目数据（要分拣的项、目标区间、抓取物…）从外部传入，**换数据即换题**，一套机制服务多个知识点。
4. **反馈铁律**：对=撒花/绿/音效；错=暖色鼓励**不用红叉**；通关=徽章掉落 + XP。
5. **难度档**：预留 easy/normal/hard（变快、目标变小、干扰变多），呼应 WarioWare 递增。
