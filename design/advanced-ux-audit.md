# 进阶课程 UX 自查与改进建议

> 目的：检查进阶课程里“交互是否匹配知识点原理”，避免换皮复用、机械点击和主题错配。  
> 结论：当前问题不是 UX 数量不足，而是部分 UX 没有按知识点类型选型。

## 总体判断

进阶课内容覆盖已经比较完整，但 UX 层不均衡。部分小节复用了通用卡片、通用点击流程或同一套地图，导致“能动”，但没有动到知识点的核心机制。

后续每个 S 小节先判断内容类型，再选 UX：

| 类型 | 应用场景 | 合适 UX |
|---|---|---|
| 机制理解型 | 卷积、注意力、反向传播、扩散去噪 | 手动操作 + 因果反馈 |
| 结构关系型 | Transformer、Agent 架构、训练流程 | 动态结构图 / 流线动画 |
| 参数探索型 | 温度、规模法则、学习率、阈值 | 滑块 + 实时曲线 |
| 了解叙事型 | AI 历史、苦涩教训、社会影响概览 | 自动动画 / 时间线 / 可暂停浏览 |
| 价值判断型 | 对齐、安全、隐私、社会风险 | 情境决策 + 后果模拟 |

## 主要问题

1. **重复组件造成疲劳**  
   `buildCompare`、`buildTree`、`buildRLTrail`、`buildScaleMilestones`、`buildForwardProp` 在不同主题里复用较多。复用本身没错，但目前一些复用只是换标题，没有体现对应知识点的独特机制。

2. **了解型内容被做成手动点击**  
   `adv:history` 是典型问题。AI 发展史属于“了解路线演变”，不是“理解一个操作因果”。让用户按顺序点亮多个年份，会变成机械劳动。更适合自动时间线、路线分叉、可拖动回看。

3. **结构型内容缺少结构图**  
   Transformer、Agent、训练三部曲这类内容，本质是“模块之间怎么流动”。如果只用普通层层点亮或卡片展开，就会弱化架构关系。这里应该让 token、工具、数据、梯度在模块之间真实流动。

4. **参数型内容没有充分利用实时探索**  
   规模法则、温度、学习率、阈值都适合滑块和曲线。只点里程碑不够，应该让孩子看到“我改一个变量，结果曲线如何变”。

5. **社会/AGI 类内容不宜硬做街机游戏**  
   这些不是反应速度问题，而是权衡和后果问题。更适合风险矩阵、情境模拟、权限边界、后果时间线。

## 逐课建议

### adv:history · AI 简史

当前问题：手动点时间线枯燥，和“了解型”不匹配。  
建议：改成**自动播放时间河流**。时间轴自动推进，分出几条路线：符号主义 → 专家系统 → 深度学习 → 大模型。用户只负责暂停、拖动、回看。  
核心 UX：`自动时间线 + 路线分叉 + 关键节点放大卡`。

### adv:scaling · 为什么越大越强

当前问题：`buildScaleMilestones` 容易像“点亮成就”，规模机制不够清楚。  
建议：做**三旋钮规模控制台**：数据量、参数量、算力三个滑块，实时影响 loss 曲线和能力灯。涌现能力不是普通节点，而是过阈值后突然亮灯。  
核心 UX：`三变量控制台 + 损失曲线 + 能力阈值灯`。

### adv:transformer · Transformer 全貌

当前问题：复用普通前向传播语义偏弱。Transformer 不是简单层层点亮，而是 token 在注意力、残差、前馈层中反复交换和加工。  
建议：做真正的**Transformer 积木剖面图**：token 输入 → 注意力交换 → 残差相加 → 前馈层加工 → 下一层。线条在模块间流动。  
核心 UX：`架构流线图 + token 行 + attention 交换动画`。

### adv:attentionmath · 注意力的数学

当前问题：只点词看连线，能表现“相关”，但数学味不够。  
建议：做 **Q/K/V 匹配器**。拖动或选择 Query token，其他词显示 Key 匹配分数，分数经过 softmax 变成权重条，最后 Value 被加权混合。  
核心 UX：`Query 选择 + score bar + softmax 权重 + Value 混合`。

### adv:rnn · RNN 与记忆

当前问题：如果只展示排队读，不能体现“隐藏状态”和“容易忘”。  
建议：做**序列传送带**。每来一个词，隐藏状态胶囊带着前文往前走；早期信息颜色逐渐变淡。讲梯度消失时，反向信号一格格缩小。  
核心 UX：`传送带 + 记忆胶囊 + 反向信号衰减`。

### adv:rl · 强化学习

当前问题：`buildRLTrail` 在试错和价值函数里重复，体验会腻。  
建议拆开三种 UX：

- 智能体试错：保留格子探索。
- 探索 vs 利用：保留老虎机，多臂赌博机很匹配。
- 价值函数：改成**热力图传播**，奖励格从亮色向周围扩散，越远价值越低。

核心 UX：`试错地图 / 多臂老虎机 / 价值热力图`。

### adv:diffusion · 扩散模型

当前去噪 UX 匹配度较高，可以保留。建议增强：

- 加噪过程用“噪声日程”滑块，显示清晰图如何逐步变成噪点。
- 文本引导改成“提示词向量拉力”，让图像在“猫/宇航员/水彩”等语义方向之间被拉动。

核心 UX：`加噪滑块 + 去噪逐步还原 + 文本引导拉力场`。

### adv:cnn · 卷积网络深入

这章相对健康，像素、卷积、池化都匹配原理。后续重点是保证每节差异明显：

- 像素矩阵：点格看 RGB。
- 卷积核：滑窗扫图找边缘。
- 特征图：同一张图经过不同卷积核生成不同响应。
- 池化：多层压缩金字塔。
- 感受野：越深层看到的区域越大。

核心 UX：`矩阵 / 滑窗 / 多特征图 / 池化金字塔 / 感受野扩张`。

### adv:classicml · 经典机器学习

KNN、聚类、阈值是目前最匹配的一组，应该保留并加强：

- KNN 加 K 值滑块，看 K=1、3、5 决策如何变化。
- 聚类加中心点移动动画，不要只瞬间分组。
- 阈值加混淆矩阵，显示分界线移动带来的误判。

核心 UX：`KNN 投票 / 聚类中心迭代 / 阈值 + 混淆矩阵`。

### adv:training · 大模型训练三部曲

当前容易变成文字流程。  
建议做**三阶段训练流水线**：预训练池 → 微调样本 → RLHF 偏好选择 → 助手输出。每阶段改变同一个模型的回答风格。  
核心 UX：`同一问题三阶段回答对比 + 训练流水线`。

### adv:society · AI 与社会

不要做普通判断题。社会议题本质是“影响扩散”。  
建议做**后果时间线**：选择“自动筛简历”“生成新闻图”“上传个人数据”等情境，看到短期好处和长期风险分叉。  
核心 UX：`情境选择 + 好处/风险双线展开`。

### adv:agi · AGI 与对齐

不适合做街机式游戏。  
建议做**目标错位模拟器**：给 AI 一个目标，比如“让房间最干净”，它可能把东西全扔掉。然后加入约束，看行为如何改变。  
核心 UX：`目标函数 + 约束开关 + 行为后果模拟`。

### adv:agentops · Agent 运行原理

应该强调循环和状态。  
建议做**Agent 控制台**：左侧任务目标，中间 ReAct 循环，右侧工具调用记录，下方记忆区。每一步高亮“想 / 做 / 看 / 记”。  
核心 UX：`ReAct 循环面板 + 工具日志 + 记忆槽`。

### adv:practical · 上手用 AI

可以更工程化：

- API：请求包从应用飞到模型，再带回答回来。
- Key：泄漏模拟器，展示别人拿到 Key 后能调用你的额度。
- Prompt：结构化提示词搭建器，目标/背景/格式/约束四块拖进去。

核心 UX：`请求响应管线 / Key 泄漏风险图 / Prompt 结构搭建器`。

## 建议新增 UX 模式库

| UX 模式 | 适合知识点 |
|---|---|
| 自动时间线 / 可拖动 scrubber | 历史、发展路线、训练阶段 |
| 架构流线图 | Transformer、Agent、API、RAG |
| Q/K/V 权重混合器 | 注意力 |
| 热力图传播 | 价值函数、影响扩散、风险传播 |
| 三变量控制台 | 规模法则、训练资源 |
| 曲线实时重绘 | 学习率、温度、阈值、过拟合 |
| 后果分叉时间线 | 社会影响、隐私、安全 |
| 目标错位模拟器 | AGI、对齐 |
| 迭代中心点动画 | 聚类 |
| 混淆矩阵反馈 | 分类阈值、公平性 |

## 改造优先级

先修最容易让用户觉得“高级课只是换皮”的 6 个：

1. `adv:history`：改自动时间线。
2. `adv:transformer`：做真实架构流线图。
3. `adv:attentionmath`：做 Q/K/V + softmax 权重。
4. `adv:rnn`：做隐藏状态传送带和梯度衰减。
5. `adv:rl`：把价值函数从重复地图改成热力图传播。
6. `adv:scaling`：把重复里程碑改成规模控制台。

保留并强化：KNN、聚类、阈值、扩散去噪、卷积滑窗、池化。这些与知识点原理高度同构，不该删，只需要补深。
