# UX 重做组件设计方案

> 目的：把 `advanced-ux-redesign-plan.md` 中的重做方向转成可实现的 HTML 互动组件规格。  
> 状态：方案稿，尚未实现。  
> 视觉约束：浅色工作台、多邻国式圆角卡片、绿/蓝/紫主色，反馈暖色不用红叉；不做重复换皮。

## 1. `uxGradientFadeTunnel`

服务：RNN S3「梯度消失」

画面：

- 一条横向 token 链：`小猫 / 昨天 / 在 / 沙发 / 睡觉`。
- 右侧输出位置有一个“错误信号”亮球。
- 点击“反向传回去”，亮球从右向左跳，每跳一格缩小 25%，透明度降低。
- 每个 token 下显示收到的梯度数值，如 `1.00 → 0.62 → 0.38 → 0.21 → 0.09`。

交互：

- 按钮 1：普通 RNN，信号快速衰减。
- 按钮 2：门控记忆，信号衰减较慢，并显示“门保留了长期信息”。

完成条件：

- 用户分别看过普通 RNN 和门控模式。

技术要点：

- 用 CSS transition 控制圆点移动和缩放。
- 数值条同步变化。
- 不用 canvas，DOM/SVG 足够。

## 2. `uxSoftmaxScale`

服务：注意力 S2「点积与 softmax」

画面：

- 左侧 Query token：`小猫`。
- 中间四个 Key 分数柱：`在 -1.1 / 沙发 2.4 / 上 0.8 / 睡觉 1.6`。
- 右侧权重条，点击 softmax 后出现百分比。
- 底部 Value 混合色块：每个 Value 用颜色和宽度表示权重。

交互：

- 用户拖动一个分数柱，softmax 权重实时重算。
- 点击“温度更低/更高”可让权重更尖锐或更平均。

完成条件：

- 用户拖动过分数，并观察权重总和保持 100%。

技术要点：

- JS 实现 softmax：`exp(score / temp) / sum(exp(...))`。
- 显示总和 `100%`，帮助理解概率分布。

## 3. `uxMultiHeadLens`

服务：注意力 S3「多头注意力」

画面：

- 中央一句话：`小猫 / 在 / 沙发 / 上 / 睡觉`。
- 顶部四个头：语法头、位置头、动作头、指代头。
- 每个头有自己的颜色。

交互：

- 点击一个头，只显示该头关注的关系线。
- 点击“合并多头”，四组关系线叠加，形成综合理解图。
- 关闭某个头，综合图少掉一类关系。

完成条件：

- 用户至少切换两个头，并点击合并。

技术要点：

- SVG viewBox 画连线，token 用 DOM 或 SVG text。
- 线条必须是主角，不能只用卡片文字说明。

## 4. `uxPositionCodeLab`

服务：Transformer S1「位置编码」

画面：

- 两句同词不同序的句子：`我 打 你` / `你 打 我`。
- 第一阶段“无位置”：token 被放进无序词袋，两句显示成相同袋子。
- 第二阶段“加位置”：每个 token 获得位置色条 `pos1/pos2/pos3`。

交互：

- 点击“打散词袋”，两句混在一起。
- 点击“加位置编码”，两个句子的路径分开，并显示不同含义。

完成条件：

- 用户完成“无位置 → 加位置”对照。

技术要点：

- 用颜色/编号强调位置是额外加进去的信息。
- 不讲公式，讲“把第几个写进向量”。

## 5. `uxTransformerBlockFactory`

服务：Transformer S2「Transformer Block」

画面：

- 工厂剖面图：Token Row → Attention → Add & Norm → FFN → Output。
- 上方一条残差旁路绕过 Attention/FFN，再加回主路。

交互：

- “走一步”依次高亮：
  1. Attention：token 之间流线亮起。
  2. Add & Norm：旁路亮起，原信息加回来。
  3. FFN：每个 token 进入独立加工槽。
  4. Output：输出送到下一层。
- “堆一层”按钮把当前 block 复制成第二层小积木。

完成条件：

- 用户走完整个 block，并堆叠一次。

技术要点：

- SVG 连线 + DOM 模块卡。
- 所有模块必须有固定尺寸，避免移动端错位。

## 6. `uxCausalMaskBoard`

服务：Transformer S3「自回归」

画面：

- 一个 6×6 attention mask 矩阵。
- 横轴是“能看见谁”，纵轴是“当前位置”。
- 允许看的格子为蓝绿，未来格子为灰锁。

交互：

- 训练模式：一次显示完整下三角 mask。
- 生成模式：点击“生成下一格”，token 从左到右出现，可见区域逐步扩大。
- 鼠标悬停某个位置，显示“当前位置不能看未来 token”。

完成条件：

- 用户生成至少 4 个 token。

技术要点：

- CSS grid 即可。
- 与词语接龙不同，这里主角是 mask，不是候选词。

## 7. `uxBitterLessonRace`

服务：大模型 S1「苦涩教训」

画面：

- 两条曲线：
  - 专家规则：起步高，快速平台期，维护成本上升。
  - 通用学习：起步低，随算力持续增长。
- 横轴为时间/算力，纵轴为能力。

交互：

- 拖动“算力时代”滑块。
- 曲线动态延伸。
- 到交叉点时出现“苦涩交叉点”：通用方法超过手写规则。

完成条件：

- 用户拖过交叉点。

技术要点：

- SVG 曲线。
- 不是胜负小游戏，而是历史趋势动画。

---

## 8. `uxScalingConsole`

服务：大模型 S2「规模法则」

画面：

- 三个资源旋钮：数据、参数、算力。
- 右侧三项输出：loss 曲线、浪费度、能力灯。

交互：

- 任意调一个滑块，实时更新：
  - 数据不足：过拟合警告。
  - 参数太小：吃不下数据。
  - 算力不足：训练不完。
  - 三者均衡：能力灯点亮。

完成条件：

- 用户调出“均衡区”。

技术要点：

- 能力不是单调取最大，而是取三者平衡后的综合分。

## 9. `uxValueHeatmap`

服务：强化学习 S3「价值函数」

画面：

- 5×5 网格，目标格是星星，坑格是黄色警告。
- 每个格子的颜色表示长期价值。

交互：

- 点击“传播价值”，价值从目标格一圈圈扩散。
- 点击任意格子，显示它的价值估计。
- 点击“加坑”，热力图绕开坑重新传播。

完成条件：

- 用户传播价值并查看至少两个格子。

技术要点：

- 用简单 Bellman 直觉：邻居价值折扣回传。

## 10. `uxDataClinic`

服务：机器学习 S4「垃圾进垃圾出」

画面：

- 左侧数据池：清晰猫、黑猫、侧脸猫、错标狗、模糊图、重复橘猫。
- 中间训练篮。
- 右侧三个仪表：标签干净度、多样性、噪声比例。
- 底部测试集：橘猫、黑猫、狗、模糊猫。

交互：

- 用户选择数据加入训练篮。
- 仪表实时更新。
- 点击训练，测试集显示预测结果。

完成条件：

- 用户训练一次坏数据和一次好数据。

技术要点：

- 不只判断“好/坏”，必须拆成数据质量维度。

## 11. `uxAICapabilityRadar`

服务：AI 大观 S1「AI 到底是不是魔法」

画面：

- 中央阿波，周围四瓣能力雷达：看、学、说、做。
- 每瓣对应输入 → 模型 → 输出。

交互：

- 点击“看”：图片 → 模型 → 标签。
- 点击“学”：例子 → 规律。
- 点击“说”：上文 → 下一个 token。
- 点击“做”：目标 → 工具调用。
- 最后四瓣合成完整 AI 能力图。

完成条件：

- 用户点亮四瓣。

技术要点：

- 用导图，不用二选一判断。

---

## 12. `uxAISystemScanner`

服务：AI 大观 S2「身边哪里有 AI」

画面：

- 生活场景：手机相册、推荐视频、智能音箱、普通台灯、闹钟。
- 每个物品点击后打开内部流程剖面。

交互：

- AI 物品显示：输入 → 模型 → 预测/生成/推荐。
- 非 AI 物品显示：开关/计时 → 固定规则 → 输出。
- 用户要判断“它为什么算/不算 AI”。

完成条件：

- 用户扫描 3 个 AI 和 2 个非 AI。

技术要点：

- 判断依据是内部机制，不是物品外观。

## 13. `uxAGIMatrix`

服务：AGI S1「通用人工智能」

画面：

- 能力矩阵：任务类型 × 能力维度。
- 三种系统切换：窄 AI / 今天的大模型 / 理想 AGI。

交互：

- 用户点击“换新任务”。
- 窄 AI 只亮一列；大模型亮多列但有空洞；AGI 理想态应跨任务稳定迁移。
- 点击“可靠性”维度，显示“大模型会做很多事，但不等于稳定可靠”。

完成条件：

- 用户切换三种系统。

技术要点：

- 避免神化 AGI，用矩阵呈现边界。

## 14. `uxReActConsole`

服务：Agent S2「ReAct」

画面：

- 四区控制台：Thought、Action、Observation、Context。
- 任务目标固定显示在顶部。

交互：

- 用户点击“下一轮”：
  1. Thought 生成计划。
  2. Action 调工具。
  3. Observation 返回结果。
  4. Context 追加观察。
- 下一轮 Thought 必须引用上一轮 Observation。

完成条件：

- 用户完成两轮，看到观察结果改变下一步。

技术要点：

- 这不是步骤流，而是上下文反馈闭环。

## 15. `uxRAGPipeline`

服务：Agent S3「记忆与 RAG」

画面：

- 文档 → 切块 → 向量库 → 问题向量 → 相似检索 → 上下文槽 → 回答。

交互：

- 点击“切块”，文档分成 4 张资料片。
- 点击“向量化”，资料片变成坐标点。
- 输入/选择问题，最近的 3 个点被吸入上下文槽。
- 点击回答，模型只基于槽内资料生成。

完成条件：

- 用户完成一次有资料回答和一次无资料回答对比。

技术要点：

- 明确“资料不是塞进模型脑子，而是塞进上下文”。

## 16. `uxAPIRequestTrip`

服务：上手用 AI S1「API」

画面：

- 前端、后端、模型服务三站。
- 一个请求包卡片显示字段：model、messages、temperature、key。

交互：

- 用户填 prompt，点击发送。
- 请求包从前端飞到后端，后端补 key 后发给模型。
- 模型返回 response：output、usage、status。
- 可切换错误参数，看 400/401 错误。

完成条件：

- 用户成功发送一次，并触发一次错误。

技术要点：

- 用飞行动画和字段展开，不用普通步骤列表。

## 17. `uxKeyLeakSimulator`

服务：上手用 AI S2「API Key」

画面：

- 左右两种架构：
  - 错误：浏览器直接带 Key 调模型。
  - 正确：浏览器请求后端，Key 只在后端。
- 费用计数器和风险提示。

交互：

- 点击“查看源码”，错误架构中 Key 高亮暴露。
- 点击“别人复制 Key”，费用计数器跳涨。
- 切到正确架构，前端源码里看不到 Key。

完成条件：

- 用户看过泄露和正确架构。

技术要点：

- 不做隐私分类，重点是工程边界。

---

## 18. `uxPromptBuilder`

服务：上手用 AI S3「提示词」

画面：

- 左侧积木块：目标、背景、资料、格式、限制、示例。
- 中间 prompt 框。
- 右侧输出稳定度仪表。

交互：

- 用户把积木块加入 prompt。
- 每加入一块，输出预览变得更稳定：
  - 只有目标：跑题。
  - 加格式：结构稳定。
  - 加示例：格式一致。
  - 加限制：更少越界。

完成条件：

- 用户搭出包含目标、格式、限制、示例的 prompt。

技术要点：

- few-shot 要用“示例锁定格式”表现，不要只写一句解释。
