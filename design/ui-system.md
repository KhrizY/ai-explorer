# AI 探险号 2.0 · UI 设计系统规范「星空实验室 Cosmos-Lab」

> 方案 A（多邻国活力）× 方案 B（深空探险）融合。**可复用模板本体 = `ui-lab/ui-system.css`**；样板间 = `ui-lab/index.html`。

## 一、核心心智模型：双表面

| 表面 | 用途 | 气质来源 |
|---|---|---|
| 🌌 **星空 `.cosmos`**（暗） | 首页地图、章节切换、庆祝/啊哈时刻、顶栏 | 方案 B：惊奇、辉光、星空 |
| 📋 **工作台 `.workbench`**（浅） | 章节内的阅读+动手区（占停留时长 80%） | 方案 A：好用、亲切、护眼 |

> 一句话：*孩子是穿越 AI 星球的探险家（暗色星空旅行），登陆站点就坐到明亮实验台前动手（浅色阅读）。* 既要 B 的酷，又不让小学生在深色里长时间阅读。

**新增约束（UX demo 同样适用）**：凡是需要停留、阅读、反复操作的页面，包括 `demos/`、`ux-lab/`、章节内交互，都归入浅色工作台。暗色星空只用于地图、顶栏、庆祝、短时黑箱效果；不要把整块游戏/可视化舞台做成长期黑底。

## 二、设计令牌（CSS variables，详见 css 文件顶部 `:root`）

- **品牌主色**：`--violet #8b5cf6`（+浅 `--violet-l` / 深 `--violet-d`），点缀 `--cyan #22d3ee`。
- **五站主题色**（同一组件换 `--_h` 即变主题）：序`--st0`琥珀 / 看`--st1`天蓝 / 学`--st2`紫 / 说`--st3`青绿 / 信`--st4`玫红。
- **反馈色（儿童安全）**：对`--correct`绿、**错`--gentle`暖琥珀（绝不用红叉）**、奖励`--gold`金。
- **圆角**：`--r-xl 28 / --r-lg 20 / --r-md 14 / --r-sm 10 / --r-pill`。
- **阴影**：`--sh-soft`柔投影 / `--sh-glow`紫辉光 / `--sh-glow-cyan`青辉光；A 风"3D 按压"用**实心偏移投影**（`0 5px 0 <深色>`）。
- **动效缓动**：`--ease-bounce`（弹跳，来自 A）/ `--ease`；统一时长 `--dur .2s`。

## 四、复用规则（给内容生产 & 后续 skill）

1. **换主题色**：任何带 `--_h` 的组件，外层设 `style="--_h:var(--st2)"` 即整体变该站主题。
2. **加交互组件**：一律套 `.ixbox` 外壳 + `.ixbox-title` + `.ixbox-hint`（👆提示），风格自动统一。
3. **反馈基调铁律**：答对必有强正反馈（撒花/绿色/音效）；答错只用 `.fb-gentle` 暖色鼓励，**禁止红叉/刺耳音**。
4. **吉祥物槽位（3.0）**：预留在工作台左下 / 反馈弹窗内，性格口吻由 #2 讲解 skill 定义，届时插入不返工。
5. **按钮可读性铁律**：按钮必须显式设置背景色、文字色和按压阴影；页面级使用 `color-scheme: light`，避免浏览器暗色模式把表单/按钮文字改黑或改白导致不可读。
6. **浅色舞台优先**：`.demo-stage`、`.gf-body`、`.ixbox` 默认浅底深字。需要强对比的网格、画布、连线，只在局部元素里用深色或高饱和色，不把整个舞台改成黑底。
7. **主色调绿蓝紫；散点用 CSS 圆点，禁用 emoji**：交互取色统一走 `--violet`/`--cyan`/`--correct`（紫·蓝·绿），暖色（金/琥珀/珊瑚）只给金币、进度、错误、边界。散点图、样本点一律用 CSS 圆点（`border-radius:50%`+`background`），**不要用 🔵🟣 等 emoji 当数据点**（依赖字体、大小不齐、缺字体不显示）；类别A/信号=蓝 `#38bdf8`、类别B/强调=紫 `#a78bfa`、画布浅底 `#eef1fb`、命中/最近邻高亮圈=橙 `#f59e0b`。
8. **组件优先复用移植，别另写简陋版**：进阶课/新课的交互，若 `demos/` 或主线已有成品，直接移植玩法并搬对应 CSS（共用本文件 css → 天然同色），别图省事重写一个不一致的版本。移植手法与"交付前真实浏览器渲染核对"详见 `viz-ux-skill.md` 第十节。
