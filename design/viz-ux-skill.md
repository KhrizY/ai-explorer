# 🔬 可视化互动模块设计 Skill

> 从打磨得最到位的三个模块（**池化 / 边缘检测滑窗 / K近邻投票**）反推出的配方。
> 用途：把任意一个抽象 AI 概念，变成一个"亲手玩几下就懂"的可视化互动模块。最终落地内容远不止 20 个，靠这套配方**举一反三批量产出**。

---

## 一、什么叫"到位"的可视化模块（判断标准）

拿三个标杆对照，它们共同做对了：

| 模块 | 抽象概念 | 变成了什么可玩的动作 | 为什么到位 |
|---|---|---|---|
| 池化 | 降采样、保留特征 | 点一下，每个 2×2 块只留最大值，逐层缩小 | **看得见"缩"的过程**，一步步 |
| 边缘检测滑窗 | 卷积找特征 | 拖小窗口扫过图，扫到边缘就变绿变亮 | **亲手扫**，因果即时（到边缘→亮） |
| K近邻 | 相似度分类 | 点空白放个新点，最近 3 个邻居高亮投票 | **自己放点**，结果当场揭晓 |

提炼出的**「到位」= 5 个必备特征**：
1. **一个抽象概念 → 一个能亲手操作的具体隐喻**（不是看，是玩）。
2. **操作驱动，非阅读驱动**：核心是"拖/点/滑一下 → 立刻看到变化"，文字只是标注。
3. **过程可见**：把抽象运算拆成**一步步 / 一层层 / 一格格**的可见动作。
4. **因果即时且夸张**：动作→结果的反馈要马上、直观、放大（高亮、变色、表情、数字跳动）。
5. **不看文字也能大致玩懂**：一句指令 + 明显的可点线索（👆/黄框/闪烁）。

## 二、设计流程（6 步，每造一个模块走一遍）

1. **抽概念的"一句话内核"**：这个概念最核心的一个动作是什么？（池化=留最强；KNN=看邻居；梯度下降=往低处走）
2. **找可操作隐喻**：把内核映射成一个孩子能"手动执行"的动作（滑窗、放点、走一步、拖滑块）。
3. **定操作方式**：拖 / 点 / 滑 / 长按 —— 选**最能体现内核**的那一种。
4. **设计"过程可见"**：怎么把运算拆成看得见的步骤/层次/高亮。
5. **加即时反馈**：变色 / 高亮 / 表情 / 数字 / 撒花，让每次操作都有"响"。
6. **诚实边界 + 参数化**：一句话点破隐喻哪里不完全对；数据/题目外部传入，一套交互复用到多个知识点。

## 四、技术实现要点（踩过的坑）

- **防网页拉伸错位**：位置判定一律用**百分比坐标 / 容器相对坐标**（`offsetX/clientWidth`），不用绝对像素+角度（黄金矿工的教训）。
- **动画**：连续动效用 `requestAnimationFrame`；纯 DOM + `transform/opacity` 优先，几何/曲线才上 `<canvas>`。
- **SVG 连线**：用 `viewBox="0 0 100 100" preserveAspectRatio="none"`，节点用 % 定位，坐标天然对齐。
- **统一外壳**：套 `.demo-stage` / `.ixbox` / `.gameframe`；反馈用 `bigFlash()`、`spark()`。
- **每个模块自带重置**：框架已统一注入 ↻。
