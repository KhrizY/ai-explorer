# 🧒 童译 Skill · 技术概念 → 9–12 岁儿童讲解 转译规范

> 本项目**统一讲解 skill**。每次写/改一节课、把一个 AI 概念讲给孩子时都走这套流程。
> 目标：不是"讲全"，是"点着"——让孩子看懂一点点、并且**还想知道更多**。

---

## 一、底层原理（调研依据，决定了下面所有规则）

1. **9–12 岁跨在"具体运算 → 形式运算"临界点**：能对**看得见摸得着**的东西做逻辑推理，但**纯符号/纯抽象**吃力（"x" 难，是因为它没有实物指代）。
2. **具体先行（concrete-first）是铁律**：先给能观察、能动手、能玩的具体体验，再过渡到抽象。
3. **类比要锚定"孩子一定经历过的具体事"**，且**显式说明"哪里不像"**——类比过度延伸会造成顽固误解（"动脉像水管"→以为血管硬且双向流）。
4. **开场用提问 + 故事 + 玩**（"你知道 X 是什么吗？"），动手体验 > 语言描述。
5. **本项目特有：UX 先行**——先想"这个概念怎么玩"，玩法定了，文字才围绕玩法写（不是写完文字再配交互）。

---

## 二、核心转译流程（6 步，每个知识点都走一遍）

```
① 抓主要矛盾 → ② 找锚点体验 → ③ 设计玩法(UX先行) → ④ 写啊哈钩子 → ⑤ 写最少的话 → ⑥ 诚实收尾
```

1. **抓主要矛盾**：这个概念对孩子**最反直觉/最难的那一个点**是什么？只攻这一点，其余全砍。（例：机器学习最难的点 = "没人告诉它答案，它怎么会的？"）
2. **找锚点体验**：把这个点映射到孩子**身边一定经历过**的具体事——分类玩具、猜谜、考试改错、认同学笔迹、玩传话游戏……
3. **设计玩法（UX 先行）**：这个锚点怎么变成**可点 / 可拖 / 可玩**的交互？先定玩法，选 `ui-system.css` 里的预制组件（滑块探索器 / 步骤流 / 黑箱…）。
4. **写啊哈钩子**：开头一个"咦？"的问题或现象，先勾起好奇，别急着解释。
5. **写最少的话**：围绕玩法写，**一句一意、先现象后原理再命名**。术语一律 `[[标记]]` 收进字典，正文只用大白话。
6. **诚实收尾**：一句话点破"这个比喻在哪不成立"（防误导），再贴术语标签。挂到心智桩。

## 三、类比引擎（怎么造一个好类比 + 校验清单）

**造法**：目标概念 → 选熟悉基底 → **显式映射**（把"A 对应 B"讲明白）→ 找边界（哪里不像）→ 用孩子试。
一个概念可用**多个类比**覆盖不同侧面。

**上线前校验清单（每个类比过一遍）**：
- ☐ 孩子**真的经历过**这个基底吗？（别用"报税""开车"这种成人经验）
- ☐ 映射的是**机制/道理**，还是只是**表面长得像**？（要前者）
- ☐ 会不会种下一个**以后很难纠正的错误印象**？
- ☐ 文化中性吗？（不依赖特定地域/家境的经验）
- ☐ 我有没有在结尾**说清它哪里不像**？

## 四、文本规则（9–12 岁）

- **句子**：一句一意，尽量短，少嵌套从句。
- **词汇**：全用日常词；专有名词一律 `[[术语]]`，释义进 `terms` 字典，**正文不留生词**。
- **密度**：每节 ≤ 5 个 block；永远"先具体、后抽象"。
- **语气**：好奇 + 鼓励 + **适当夸张/幽默/搞怪**；用第二人称"你"；像朋友不像老师。
- **数字/名词**：不堆年份、不堆公司名、不报参数（除非对理解真有帮助）。

## 五、4 个心智模型桩（每段都要往上挂至少一个）

- 🧲 不是魔法，是从例子找规律
- 🍎 垃圾进 → 垃圾出（数据决定聪明程度）
- 🎭 它不"懂"，它在猜/配对
- ⚖️ 会犯错、有好有坏，要会怀疑

## 六、反模式（踩坑清单，写完自查）

- ❌ 先讲抽象定义，再举例子（顺序反了）
- ❌ 一节塞进 3 个以上新概念
- ❌ 类比过度延伸、不说边界
- ❌ 只有文字、没有可玩的交互
- ❌ 术语混进正文当生词
- ❌ 答错用红叉/说教（要暖色鼓励）
- ❌ 用成人经验做类比

## 七、吉祥物人设槽位（3.0 预留，但**现在写文本就按它的口吻**）

- **角色设定**：一个叫「阿波」的 AI 讲解员机器人。性格：**超级好奇、爱鼓励、偶尔犯傻自嘲**——它自己也会算错、会认错猫，正好用来演示 🎭🍎⚖️（"看，连我也会搞错！"）。
- **口吻库槽位**（写文本时，这几类语句都按阿波的口吻写，3.0 插入吉祥物即无缝）：
  - 开场钩子语 / 答对欢呼 / 答错鼓励 / 小节过渡引导 / 彩蛋搞怪语
- 现阶段：文本里的 `nextHint`、反馈语、钩子问句，都用这个"好奇爱鼓励"的口吻，**不出现"本节将介绍"式教材腔**。

## 九、一个完整转译样例（"训练"概念，演示 6 步）

1. **主要矛盾**：没人给答案，它怎么变准的？
2. **锚点体验**：做错题→对答案→改→下次对。
3. **玩法**：「训练循环手摇机」——点一下走一步，看准确率从瞎猜爬到很准。
4. **钩子**："阿波刚出生时，认猫全靠瞎猜。它是怎么一步步变成认猫高手的？点点看👇"
5. **最少的话**："每次它猜完，我们告诉它对不对，它就悄悄[[调整]]自己一点点。猜几万次，就从瞎猜变很准了。"
6. **诚实收尾**："注意哦——它不是'想通了'，只是把内部的数字改得越来越合适。这不是魔法🧲，是大量重复。"
