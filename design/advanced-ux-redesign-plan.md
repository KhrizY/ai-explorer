# 进阶与主线局部 UX 重做方案

> 记录日期：2026-07-15  
> 目的：把目前“和知识点内容无关的生搬硬套 UX”标记出来，重新设计成真正服务理解的 HTML 互动 / 动画 / 可视化。  
> 原则：不先动实现；先定每节的知识点内核和表现方案。每个重做 UX 必须不同，不能只换皮复用。

## 0. 总原则

本轮重做不是“多加小游戏”，而是让每个知识点有自己的可视化语法。

- **机制型知识点**：做机制动态重现，让内部动作看得见。
- **参数型知识点**：做滑块/控制台，实时显示变量改变后的结果。
- **结构型知识点**：做导图/架构流线，让模块关系和信息流可见。
- **流程型知识点**：做流水线/状态机，让输入、处理、输出、反馈闭环可见。
- **概念边界型知识点**：做对照矩阵/能力雷达，不做反应速度游戏。

上线前检查：

- 这个 UX 的核心动作是否等同于知识点的核心机制？
- 删除说明文字后，孩子能不能从动画/交互里看出“它在干什么”？
- 是否和前后章节重复？如果只是换图标/换题目，必须重做。
- 是否保持浅色工作台、多邻国式圆润 UI、绿蓝紫主色？

## 1. 必须重做清单与方案

### RNN · S3「梯度消失 · 长句为什么容易忘」

当前问题：现有 UX 复用“排队读/新旧办法对比”，只能说明 RNN 顺序读，不能说明梯度消失。

知识点内核：错误信号从后往前传，越传越弱，前面的词学不到。

新 UX：**反向信号衰减隧道**

- 画一条句子链：`小猫 / 昨天 / 在 / 沙发 / 睡觉`。
- 用户点击“从最后一个词传回错误”。
- 亮色脉冲从右往左走，每过一格缩小、变淡、数值降低。
- 到最左侧时几乎消失，提示“开头的词收到的学习信号太弱”。
- 再切换“LSTM/GRU 门控”模式：记忆门打开后，早期信息保留得更久。

不重复点：不是读句子，也不是普通连线，而是专门表现“反向传播信号衰减”。

---

### 注意力数学 · S2「点积与 softmax · 分数变权重」

当前问题：复用注意力连线，只能看到“相关”，看不到分数如何变成权重。

知识点内核：Query 和 Key 点积得到分数；softmax 把分数变成总和为 1 的权重；权重决定 Value 混合比例。

新 UX：**softmax 权重秤**

- 左侧选 Query token。
- 中间显示每个 Key 的点积分数柱：可正可负。
- 点击“softmax”后，分数柱流入一个“权重秤”，变成百分比条，总和固定 100%。
- 用户拖高某个分数，看到 softmax 后高分权重被放大、低分被压低。
- 右侧显示 Value 混合结果：不同颜色的 token 信息按权重混合成输出。

不重复点：这不是词间连线，而是“分数 → 概率权重 → 加权混合”的数学过程。

---

### 注意力数学 · S3「多头注意力 · 多种关系一起看」

当前问题：仍复用单组注意力，无法说明“多头”为什么必要。

知识点内核：不同头学习不同关系子空间；同一句话可以同时看主谓、位置、指代、修饰等关系。

新 UX：**多头关系透镜**

- 同一句句子放在中央：`小猫 / 在 / 沙发 / 上 / 睡觉`。
- 顶部有 4 个头：主语头、位置头、动作头、指代头。
- 点击不同头，显示不同颜色的关系线。
- 点击“合并多头”，四层半透明关系线叠成一张综合理解图。
- 可加一个“只开单头”对照：单头只能看一种关系，多头能同时保留多种关系。

不重复点：不是重复 Q/K/V，而是表现“多个关系视角并行”。

---

### Transformer · S1「位置编码 · 同时看也要知道顺序」

当前问题：复用“排队读 vs 一眼看”，只能说明并行，不能说明位置编码如何补顺序。

知识点内核：Transformer 并行看 token，本身没有顺序感；位置编码把“第几个”写进 token 表示里。

新 UX：**无序词袋 vs 加位置标签**

- 先把一句话 token 打散成词袋：`我 / 打 / 你` 和 `你 / 打 / 我` 混在一起。
- 让模型只看 token，不看位置，两个句子会被画成相近的“混乱袋子”。
- 点击“加位置编码”，每个 token 贴上位置条纹/坐标。
- 句子向量立刻分成两条不同路径，结果含义变成“我打你”/“你打我”。

不重复点：不是读句速度对比，而是专门表现“位置信息改变语义”。

---

### Transformer · S2「Transformer Block · 注意力 + 前馈层」

当前问题：复用普通前向传播，和 Transformer block 的结构不匹配。

知识点内核：一个 block 里先做 token 间信息交换，再逐 token 加工；残差和归一化保证深层稳定。

新 UX：**Transformer 工厂剖面图**

- 一排 token 进入工厂。
- 第一站 Attention：token 之间出现流线，互相取信息。
- 第二站 Add & Norm：原输入绕旁路加回来，画成“保底旁路”。
- 第三站 FFN：每个 token 单独进入小加工机，颜色/形状被精炼。
- 点击“堆叠下一层”，同一流程缩小成一块 block，向上叠第二层。

不重复点：结构图不是普通神经网络层，而是完整表现 Attention、残差、归一化、FFN 的职责分工。

---

### Transformer · S3「自回归 · 一个 token 一个 token 生成」

当前问题：词语接龙能说明“猜下一个”，但没有表现 causal mask 和生成时不可并行。

知识点内核：训练时可并行预测多个位置；生成时每次只能用已生成 token 预测下一个，未来 token 被遮住。

新 UX：**因果遮罩生成棋盘**

- 横轴是 token 位置，纵轴是“能看见谁”。
- 训练模式：显示下三角 causal mask，每个位置只能看左边和自己。
- 生成模式：token 从左到右一个个亮起，未来格子保持灰色锁住。
- 用户点击“生成下一格”，下一个 token 出现，mask 可见区域扩大一列。

不重复点：不是普通接龙，而是表现“为什么自回归只能从左到右生成”。

---

### 大模型 / 规模法则 · S1「苦涩教训 · 通用方法赢在长期」

说明：这里暂按 `adv:scaling` 的“大模型为什么越大越强”处理。

当前问题：比较卡片太泛，不能说明“能吃算力的通用方法长期胜出”。

知识点内核：手写规则短期有效，但扩展困难；通用方法随着数据和算力增长持续变强。

新 UX：**两条成长曲线赛跑**

- 左边“专家规则”曲线起步高，但很快进入平台期，并出现维护成本条越堆越高。
- 右边“通用学习”曲线起步低，随着数据/算力滑块增长持续上升。
- 用户拖动时间/算力轴，看两条曲线何时交叉。
- 交叉点标注“苦涩”：聪明规则输给可扩展方法。

不重复点：不是单纯比较两张卡，而是把“长期可扩展性”变成曲线交叉。

---

### 大模型 / 规模法则 · S2「规模法则 · 数据参数算力三件套」

当前问题：单滑块点亮能力门槛，过于简化。

知识点内核：数据、参数、算力要匹配增长；单独堆一个会浪费或过拟合。

新 UX：**三资源配平控制台**

- 三个滑块：数据量、参数量、算力。
- 实时输出三条指标：训练 loss、浪费度、能力指数。
- 如果参数远大于数据，出现“过拟合风险”；如果数据远大于参数，出现“模型吃不下”；如果算力不足，出现“训练不完”。
- 三者平衡时曲线最稳，能力灯逐个亮起。

不重复点：不是“规模越大越好”，而是“配平才有效”。

---

### 强化学习 · S3「价值函数 · 哪个状态更值得去」

当前问题：复用踩格子探路，和 S1 试错重复，不能解释价值函数。

知识点内核：状态价值是对未来累计奖励的估计；奖励会从目标状态向前传播。

新 UX：**价值热力图回传**

- 5×5 网格，右下角有奖励星星。
- 点击“价值传播”，星星周围先变绿，再一圈圈向外扩散，颜色逐渐变浅。
- 用户点击任意格子，显示“当前位置价值 = 未来可能拿到奖励的估计”。
- 加一个坑格后，热力图绕开坑，体现价值不是距离，而是长期收益。

不重复点：S1 是行动试错；S3 是价值估计传播。

---

### 机器学习 · S4「垃圾进垃圾出 · 数据决定聪明」

当前问题：当前“选数据→训练→评判”方向正确，但仍偏小游戏，没有充分说明数据分布、标签错误、覆盖不足三类问题。

知识点内核：数据质量不只是“好/坏”，包括标签准确性、分布覆盖、多样性、噪声。

新 UX：**训练数据诊断台**

- 左侧是数据池，分为：正确猫图、错标狗图、模糊图、单一橘猫、黑猫/侧脸猫。
- 用户拖入训练篮。
- 右侧实时出现三个诊断仪表：
  - 标签干净度
  - 多样性覆盖
  - 噪声比例
- 点击训练后，测试集展示：橘猫、黑猫、背影猫、狗。模型表现与三项指标对应。

不重复点：不是单纯选好坏，而是让孩子看懂“数据质量由多个维度组成”。

### AI 大观 / 序章 · S1「AI 到底是不是魔法」

说明：这里暂按主线 `序` 的 S1 处理。

当前问题：如果只是判断“魔法/学来的”，容易停留在态度判断，没有形成 AI 总览。

知识点内核：AI 不是魔法，而是一组能力：感知、学习、生成、行动；每种能力背后都有输入、模型、输出。

新 UX：**AI 能力雷达总览**

- 中央是阿波，周围四个能力雷达瓣：看、学、说、做。
- 用户点击每瓣，看到对应输入输出：
  - 看：图片 → 标签
  - 学：例子 → 规律
  - 说：文字 → 下一个 token
  - 做：目标 → 工具调用
- 最后四瓣合成“AI 不是魔法，是很多计算能力拼起来”。

不重复点：不是普通二选一，而是开场总览导图。

---

### AI 大观 / 序章 · S2「身边哪里有 AI」

当前问题：找 AI 物品如果只靠点击，会像普通观察游戏，解释力不足。

知识点内核：判断一个系统是不是 AI，要看它是否在做感知、预测、推荐、生成、决策，而不是看它是不是电子产品。

新 UX：**身边物品剖面扫描**

- 画一个生活场景：手机、闹钟、相机、推荐视频、智能音箱、普通台灯。
- 用户点击物品，弹出“内部流程剖面”：
  - 普通台灯：开关 → 电路 → 亮
  - 相册识猫：图片 → 模型 → 猫标签
  - 推荐视频：历史行为 → 模型 → 推荐列表
- 不是 AI 的物品也要显示“没有学习/预测，只是固定规则”。

不重复点：不是找图标，而是用流程剖面解释“为什么它算/不算 AI”。

---

### AGI · S1「AGI · 通用人工智能是什么」

当前问题：比较卡片过泛，不能说明 AGI 与今天大模型的边界。

知识点内核：AGI 不是“会很多任务”这么简单，还涉及跨任务迁移、持续学习、自主规划、可靠性。

新 UX：**能力迁移矩阵**

- 横轴：任务类型，语言、图像、数学、机器人、长期计划。
- 纵轴：能力维度，理解、学习、迁移、执行、可靠。
- 今天的大模型显示为部分格子亮，AGI 理想状态显示为更多格子稳定亮。
- 用户点击“换新任务”，普通窄 AI 只在一个格子亮，大模型迁移一部分，AGI 理想状态应跨域适应。

不重复点：不是二选一比较，而是展示 AGI 的多维条件和现实缺口。

---

### Agent · S2「ReAct · 想做看再想」

当前问题：步骤流太像普通流程卡，没有表现“观察结果会改变下一步”。

知识点内核：ReAct 是 Reasoning + Acting 的闭环；每轮 observation 写回上下文，影响下一轮计划。

新 UX：**ReAct 调试控制台**

- 四个区域：Thought、Action、Observation、Context。
- 用户给目标：“查明今天适不适合出门”。
- 第一步 Thought 生成搜索计划，Action 调天气工具，Observation 返回“下雨”。
- Context 区自动追加 observation。
- 下一轮 Thought 改成“建议带伞/改室内活动”。
- 用户可故意选择错误工具，看 observation 如何迫使下一轮修正。

不重复点：不是线性步骤，而是“结果写回上下文再改变思考”的闭环。

---

### Agent · S3「记忆与 RAG · 把资料接进来」

当前问题：RAG 演示偏“先查资料再回答”，还不够体现检索、向量相似、资料拼入上下文。

知识点内核：RAG = 文档切块 → 向量化 → 相似检索 → 拼入上下文 → 生成回答。

新 UX：**RAG 资料管道剖面**

- 左侧一篇长文档被自动切成资料块。
- 点击“向量化”，资料块变成彩色坐标点进入向量库。
- 用户输入问题，问题也变成一个点。
- 最近的 3 个资料块被吸出来，放进“上下文槽”。
- 模型只基于上下文槽回答；如果槽为空，则回答变模糊或提示不知道。

不重复点：不是普通查资料，而是可视化“相似检索 + 上下文注入”。

### 上手用 AI · S1「API · 应用怎样调用模型」

当前问题：点步骤太线性，不能体现请求包、服务端、模型响应、错误处理。

知识点内核：API 调用是应用向模型服务发送结构化请求，服务验证身份后返回响应。

新 UX：**请求包旅行图**

- 用户在左侧填一条 prompt。
- 点击发送，请求包显示字段：model、messages、temperature、key。
- 包从前端飞到后端，再到模型服务。
- 模型服务返回 response 包，字段展开显示 output、usage、status。
- 如果缺 key 或参数错，包被拦截，显示 401/400 错误。

不重复点：不是“走一步流程”，而是让请求/响应结构可见。

---

### 上手用 AI · S2「Key · 身份凭证不能乱放」

当前问题：隐私分拣不适合讲 API Key；Key 是工程凭证，不是普通个人信息分类。

知识点内核：API Key 是调用权限；放前端会暴露；正确做法是后端代理。

新 UX：**Key 泄漏模拟器**

- 左侧有两种架构：
  - 错误：浏览器 → 直接带 Key 调模型
  - 正确：浏览器 → 自家后端 → 后端带 Key 调模型
- 点击“查看网页源码”，错误架构里 Key 直接暴露。
- 点击“别人复制 Key”，费用计数器开始上涨。
- 切到正确架构后，前端看不到 Key，只能请求后端受控接口。

不重复点：不是隐私守卫，而是工程安全架构图。

---

### 上手用 AI · S3「提示词 · 任务、资料、格式、边界」

当前问题：三锦囊卡片太泛，不能教会结构化 prompt。

知识点内核：好提示词由任务、背景、约束、格式、示例组成；few-shot 用例子约束输出形态。

新 UX：**Prompt 结构搭建器**

- 左侧有积木块：目标、背景、资料、格式、限制、示例。
- 用户拖/点块到提示词框。
- 右侧实时显示模型输出稳定度仪表：
  - 只写目标：输出飘
  - 加格式：结构稳定
  - 加示例：格式更一致
  - 加限制：少跑题
- 可点击“few-shot 示例”，看到输出格式被样例锁住。

不重复点：不是建议卡，而是让用户亲手搭出 prompt 结构。

## 2. 重复课程自查

### CNN 深入 vs 主线「看」

主线「看」已经覆盖：

- 像素：图片是数字。
- RGB：颜色由三个数字组成。
- 卷积：小窗口扫描找特征。
- 层级特征：从线条到物体。
- 池化：缩小图，留住重点。
- 波形：声音也是数字。

进阶 `adv:cnn` 当前覆盖：

- 像素矩阵。
- 卷积核。
- 池化。

重复度判断：**高度重复**。目前 `adv:cnn` 不是“深入”，只是主线「看」的短版复述。继续保留会稀释进阶区质量。

建议处理：

1. **短期：从进阶列表移除 `adv:cnn`**，避免重复学习路径。
2. **若未来保留，必须改成真正进阶版**，主题应改为：
   - 多卷积核生成多张特征图。
   - padding / stride 如何改变输出尺寸。
   - 感受野如何随层数扩大。
   - 参数共享为什么减少参数。
   - CNN 对平移更鲁棒，但对旋转/遮挡仍有限。
3. 如果不做这些，就不要保留 CNN 深入课。

结论：**建议先移除，未来有足够新内容再恢复。**

---

### AI 与社会 vs 主线「信」

主线「信」已经覆盖：

- AI 会偏心。
- 不要盲信 AI。
- AI 在猜，不知道对错。
- 隐私保护。
- 好用途 / 坏用途。
- 聪明使用 AI 的基本态度。

进阶 `adv:society` 当前覆盖：

- 自动化与工作变化。
- 偏见与公平。
- 深度伪造。

重复度判断：**部分重复**。

- `偏见与公平` 与主线「信」S1 高度重复。
- `深度伪造` 与主线「信」的“好坏用途/别盲信”有中度重复，但如果讲媒体生态和生成内容溯源，可以保留。
- `自动化与工作变化` 是新主题，可保留。

建议处理：

1. **不要保留泛泛 AI 道德课**，尤其不要再讲“要公平、要保护隐私、别盲信”这类主线已讲内容。
2. 如果保留 `adv:society`，应改成“AI 社会系统与影响扩散”，而不是“AI 道德复习”：
   - 自动化如何替代任务而不是整个职业。
   - 生成内容如何改变信息传播链。
   - 平台标注、水印、来源核验如何进入内容生态。
   - 算法推荐如何形成反馈循环。
3. `偏见与公平` 如果保留，必须改成工程化视角：
   - 分群评测矩阵。
   - 总准确率 vs 群体准确率。
   - 数据审计和人工复核流程。

结论：**AI 与社会不建议整体删除，但必须去掉泛道德重复内容，改成系统影响/传播机制/工程治理。**
